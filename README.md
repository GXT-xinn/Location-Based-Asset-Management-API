# Asset Condition Assessment API
<a name="top"></a>
A technical guide for developping a simple application server to handle data request for the [Asset Condition Assessment App](https://github.com/ucl-geospatial-21-22/cege0043-apps-21-22-GXT-xinn). This server body is stored in ```dataAPI.js``` file. This RESTful API provides the functionality to interact with the PostgreSQL database, including save, remove, retreive data as GeoJSON file from the database and return confirm receipts when interaction completed. Funtions to support those interactions are stored separatly in the ```crud.js``` file for debugging purpose.

### 2. Deployment

* **Procedures to deploy this application server**:

1. Clone the source code of this api from Github to CEGE server at ```home/studentuser/code``` by typing in the command line (terminal) window for Ubuntu: 

```
$ cd /home/studentuser/code
$ git clone https://github.com/ucl-geospatial-21-22/cege0043-api-21-22-GXT-xinn.git
```

2. Create a file called ```postGISConnection.js``` to store the user information for database connection, and save the file to ```/home/studentuser/certs``` directory, **which you will need to create the folder first**, format like the following:

```javascript
host: *XXX*,
user: *userXXX*,
database: *ucfscde*,
password: *cege0052XXXXXX*,
port: *5432*
```

3. Download and install the Node.js

```
$ cd /home/studentuser/code/cege0043-api-21-22-GXT-xinn
$ npm install express
```

4. Download and install the pg packages

```
$ cd /home/studentuser/code/cege0043-api-21-22-GXT-xinn
$ npm install pg -save
```


5. Make sure the Node JS server is successfully started. If any error occurs, you could enter the debug mode through the command line window by typing

```
$ cd /home/studentuser/code/cege0043-api-21-22-GXT-xinn
$ node dataAPI.js
```

## 4. File description

The files associated to this RESTful API are located in the ```cege0043-api-21-22-GXT-xinn``` folder and its subfolders ```~/routes```.
* ```~/cege0043-api-21-22-GXT-xinn```
	* ```dataAPI.js```: 
		- references the packages and codes
		```javascript
		var express = require('express');
		var path = require("path");
		var fs = require('fs');
		var http = require('http');
		var app = express();
		var httpServer = http.createServer(app);
		```
		- Assigns 4480 aa the listening port
		```javascript
		httpServer.listen(4480);
		```
		- Return designated message when receives root level request
		```javascript
		app.get('/',function (req,res) {
			res.send("Hello World from the class test server");
		});
		```
		- Function to support CROSS ORIGIN (CORS) requests
		```javascript
		app.use(function(req, res, next) { 
			res.setHeader("Access-Control-Allow-Origin", "*"); 
			res.header("Access-Control-Allow-Headers", "Origin, X-Requested-With, Content-Type, Accept"); 
			res.setHeader('Access-Control-Allow-Methods', 'GET,PUT,POST,DELETE'); 
			next(); 
		});
		```
		- Function to log the CROSS ORIGIN requests in the linux server for debugging 
		```javascript
		app.use(function (req, res, next) { 
			var filename = path.basename(req.url); 
			var extension = path.extname(filename); 
			console.log("The file " + filename + " was requested."); 
			next(); 
		});
		```
		- Registers ```crud.js``` file in order to use the funtionalies included
		```javascript
		const crud = require('./routes/crud');
		app.use('/', crud);
		```
		


	* ```~/routes```
		* ```crud.js```:  CRUD stands for Create, Read, Update, Delete as to explicily describe the main funtionality of this files. Similar to the ```dataAPI.js```, this file need to requires several packages and codes (e.g. express, pg, fs, os body-parser) to support the request's operation.
		
			* **Locate the database login information**:
			While the database login information was created and stored in the ```/home/studentuser/certs/postGISConnection.js``` as required in the deployment section.
			Details in the configruation file will be converted into the pair array format to successfully connnect to the database using a connection ```pool```.
			
			* **Main functionalities**:
		
			|       request  					|    type of request   |	  description     |
			|  -------------------------------- |    --------------    | 	--------------    |
			|~/getUserId         				|  GET requests[*](#get).  |	This request gives the id number corresponding to your database login name in the configruation file.  |
			|~/insertConditionInformation       |  POST requests[*](#post).  |	This request inserts information entered through asset form to the database, including asset's name, installation date, and location.  |
			|~/insertAssetPoint         		|  POST requests.  |	This request inserts information entered through condition form to the database, including asset's name and condition description. |
			|~/geoJSONUserId/:user_id         	|  GET requests.  | This request gives asset information in a GeoJSON format for a specific user ID.  |
			|~/userConditionReports/:user_id    |  GET requests.  |	This request tells user how many condition reports they have saved in a json array format, when they add a new condition report  |
			|~/userRanking/:user_id         	|  GET requests.  |	This request tells users their ranking based on condition reports in a json array format, in comparison to all other users. |
			|~/fiveClosestAssets/:latitude/:longitude  |  GET requests.  |	This request gives 5 assets closest to the user’s current location, added by any user, in GeoJSON format. |
			|~/lastFiveConditionReports/:user_id       |  GET requests.  |	This request gives the last 5 reports that the user created (colour coded depending on the conditition rating) in GeoJSON format. |
			|~/conditionReportMissing/:user_id         |  GET requests.  |	This request gives a list of assets and calculates proximity alerts for assets that the user hasn’t already given a condition report for in the last 3 days in GeoJSON format. |
			|~/assetsInGreatCondition         	|  GET requests.  |	This request gives list of all the assets with at least one report saying that they are in the best condition in json array format.  |
			|~/dailyParticipationRates         	|  GET requests.  |	This request gives the daily reporting rates for the past week, categorized into two types: reports_submitted and reports_not_working (in worst condition), in json array format. |
			
			<a name="get"></a>
			* GET requests: The request can be called from a browser URL.
			<a name="post"></a>
			* POST requests: The request can only be called from AJAX.
